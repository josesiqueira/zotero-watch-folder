# Phase 1 runbooks — Core watch folder

Covers basic config, import flow, post-import actions, file-type filter, import modes, first-run, and bidirectional deletion sync. Assumes SMOKE.md S.1 has already configured the watch folder.

---

## 1.1 — Pref pane renders all fields

**MCP-automatable:** manual-only (visual)
**Steps:** open `Edit → Settings → Watch Folder`. Visually confirm: icon, Enable checkbox, Source Folder + Browse, Target Collection, Poll Interval, File Types, Import Mode dropdown, Post-Import Action dropdown, Auto-Retrieve checkbox, Auto-Rename checkbox, Rename Pattern.
1. `zotero_open_preferences { paneId: "watch-folder@zotero-plugin.org" }` — pane opens without errors.
2. `zotero_read_errors { lines: 10 }` — no XHTML errors.

**Pass criteria:** all fields visible, no console errors.

## 1.2 — Enable from scratch

**MCP-automatable:** full
**Steps:**
1. `zotero_set_pref { key: "extensions.zotero.watchFolder.sourcePath", value: "/tmp/ZoteroWatchTest/inbox" }`
2. `zotero_set_pref { key: "extensions.zotero.watchFolder.targetCollection", value: "TestImports" }`
3. `zotero_set_pref { key: "extensions.zotero.watchFolder.pollInterval", value: 5 }`
4. `zotero_set_pref { key: "extensions.zotero.watchFolder.enabled", value: true }`
5. `zotero_plugin_reload { pluginId: "watch-folder@zotero-plugin.org" }`
6. `zotero_read_logs { filter: "WatchFolder", lines: 40 }` — expect `Plugin started successfully` and `Started watching folder`.

**Pass criteria:** logs confirm watch started.
**Cleanup:** none (needed for 1.3+).

## 1.3 — Detect and import a PDF

**MCP-automatable:** partial

3. `zotero_db_query { sql: "SELECT itemID FROM items WHERE dateAdded > datetime('now','-2 minutes')" }` — ≥ 1 row.
4. Bash: confirm `/tmp/ZoteroWatchTest/inbox/sample.pdf` still exists (default action = `leave`).

**Pass criteria:** item exists, source PDF still present.
**Cleanup:** none yet.

## 1.4 — Metadata retrieval populates fields

**MCP-automatable:** partial
**Steps (after 1.3):**
1. Wait up to 60s.
2. `zotero_read_logs { filter: "metadata|Recognition", lines: 80 }` — expect `Queued item N for metadata retrieval` AND a matching `Recognition completed` or `timed out`.
3. `zotero_execute_js { code: "const items=await Zotero.Items.getAll(Zotero.Libraries.userLibraryID); const it=items.slice(-1)[0]; return { title:it.getField('title'), doi:it.getField('DOI'), creators:it.getCreators().length, tags:it.getTags().map(t=>t.tag) };" }`

**Pass criteria:** either title + creators populated, or tag `_needs-review` present. The **absence of a matching `Recognition completed/timed out` log** while a `Queued` log exists means the fire-and-forget bug in `metadataRetriever._processQueue()` swallowed the result — report as failure.

## 1.6 — Post-import action: delete

**MCP-automatable:** full
**Steps:**
1. `zotero_set_pref { key: "extensions.zotero.watchFolder.postImportAction", value: "delete" }`
2. Bash: `cp /path/to/sample2.pdf /tmp/ZoteroWatchTest/inbox/`. Wait one poll cycle + 5s.
3. `zotero_read_logs { filter: "Deleted source file|postImport", lines: 60 }` — expect deletion log.
4. Bash: `test ! -f /tmp/ZoteroWatchTest/inbox/sample2.pdf && echo gone` → `gone`.
5. `zotero_execute_js { code: "const f=PathUtils.join(Zotero.DataDirectory.dir,'zotero-watch-folder-tracking.json'); const txt=await IOUtils.readUTF8(f); const j=JSON.parse(txt); const rec=Object.values(j.records||{}).find(r=>r.path && r.path.endsWith('sample2.pdf')); return rec && rec.expectedOnDisk === false;" }` → `true`.

**Pass criteria:** file removed; tracking record has `expectedOnDisk:false`.
**Cleanup:** `zotero_set_pref { key: "extensions.zotero.watchFolder.postImportAction", value: "leave" }`.

## 1.7 — Post-import action: move (recursive subfolder mirror)

**MCP-automatable:** full
**Steps:**
1. `zotero_set_pref { key: "extensions.zotero.watchFolder.postImportAction", value: "move" }`
2. Bash: `mkdir -p /tmp/ZoteroWatchTest/inbox/Research/AI && cp /path/to/sample3.pdf /tmp/ZoteroWatchTest/inbox/Research/AI/`. Wait poll + 5s.

5. `zotero_execute_js { code: "const root=await Zotero.Collections.getByLibrary(Zotero.Libraries.userLibraryID); const find=n=>root.find(c=>c.name===n); return { research:!!find('Research'), ai:!!find('AI') };" }` — both `true` (recursive collection path created under `TestImports`).

**Pass criteria:** file moved into `imported/` subfolder mirror; collections `Research` and `AI` were auto-created.
**Cleanup:** reset action to `leave`; remove the moved file.

## 1.11 — Zotero → disk deletion sync (3-button dialog)

**MCP-automatable:** manual-only (right-click + dialog). Run with `diskDeleteOnTrash = ask`.

Reset before each sub-case: `zotero_set_pref { key: "extensions.zotero.watchFolder.diskDeleteOnTrash", value: "ask" }`.

### 1.11.a — Move to OS trash (default)
1. Trash an item in Zotero. In dialog: `Move to OS trash`.
2. `zotero_read_logs { filter: "Trash sync", lines: 40 }` → `moved <path> to OS trash`.
3. Bash: confirm file is in OS trash (Linux: `~/.local/share/Trash/files/`; macOS: `~/.Trash/`).
4. `zotero_db_query { sql: "SELECT deleted FROM items WHERE itemID=<ID>" }` → `1`.

### 1.11.b — Keep on disk
1. Trash another item. Click `Keep on disk`.
2. `zotero_read_logs { filter: "Trash sync", lines: 40 }` → NO `moved` / `deleted` line.
3. Bash: file still present in inbox.
4. Item is in Zotero Bin (`deleted=1`).

### 1.11.c — Delete permanently
1. Trash another item. Click `Delete permanently`.
2. `zotero_read_logs { filter: "Trash sync", lines: 40 }` → `permanently deleted <path>`.
3. Bash: file gone AND not in OS trash.

### 1.11.d — Don't ask again persists choice
1. Trash item, tick `Don't ask again`, click `Move to OS trash`.
2. `zotero_get_pref { key: "extensions.zotero.watchFolder.diskDeleteOnTrash" }` → `os_trash`.
3. Trash another item — no dialog should appear; action is taken silently.
4. Repeat for `Keep on disk` → pref becomes `never`; `Delete permanently` → pref becomes `permanent`.

### 1.11.e — Linked-mode warning
1. With `importMode=linked`, import a fresh file, then trash it.
2. `zotero_screenshot { target: "main-window" }` — dialog text mentions `linked` and `ONLY copy`.
3. Click `Delete permanently`; verify file gone.

### 1.11.f — Multi-item batched prompt
1. Reset to `ask`. Select 3+ items, trash in one operation.
2. `zotero_screenshot { target: "main-window" }` — single dialog with `N items were moved to Zotero's bin.`
3. Choose `Move to OS trash`. All N files end up in OS trash.

**Cleanup:** restore items from Bin as needed; reset `diskDeleteOnTrash` to `ask`; reset `importMode` to `stored`.

## 1.12 — Disk → Zotero deletion sync (external delete)

**MCP-automatable:** partial. Run with `diskDeleteSync = auto`.

### 1.12.a — Single external delete
1. `zotero_set_pref { key: "extensions.zotero.watchFolder.diskDeleteSync", value: "auto" }`
2. Bash: `rm /tmp/ZoteroWatchTest/inbox/<tracked>.pdf`. Wait one poll + 5s.
3. `zotero_read_logs { filter: "externally-deleted", lines: 40 }` → `Detected 1 externally-deleted file(s)`.
4. `zotero_screenshot { target: "main-window" }` — popup lists the path and item title.
5. `zotero_db_query { sql: "SELECT deleted FROM items WHERE itemID=<ID>" }` → `1`.

### 1.12.b — Batched popup for multiple deletes
1. Bash: `rm` three tracked files in one shell call. Wait one poll + 5s.
2. `zotero_read_logs { filter: "externally-deleted", lines: 40 }` → `Detected 3 externally-deleted file(s)`.
3. Popup lists up to 20 entries with `…and N more` if exceeded.

### 1.12.c — diskDeleteSync=never disables
1. `zotero_set_pref { key: "extensions.zotero.watchFolder.diskDeleteSync", value: "never" }`
2. Bash: `rm` another tracked file. Wait one poll + 5s.
3. `zotero_read_logs { filter: "externally-deleted", lines: 40 }` → no log.
4. `zotero_db_query { sql: "SELECT deleted FROM items WHERE itemID=<ID>" }` → `0`.

### 1.12.d — Linked-mode popup wording
1. Reset to `auto`. `zotero_set_pref { key: "extensions.zotero.watchFolder.importMode", value: "linked" }`. Import a fresh linked file, then `rm` it.
2. Popup must mention `linked attachments` and `broken file links`.

**Cleanup:** reset `diskDeleteSync` to `auto`, `importMode` to `stored`. Restore items from Bin if needed.
